Product Routes

POST
/product
{ name: { type: String, required: true, }, category: { type: String, required: true, }, images: { max: 2, required: true, type: Array, allowedTypes: ['jpg','png', 'gif', 'jpeg'], }, description: { type: String, required: true }, price: { type: String, required: true }, totalQuantity: { type: Number, required: true }, available: { type: Boolean, // Either true or false. Tells if a product is available or out of stock default: true }, variations: { type: Object, default: {} // Empty object meaning this product has no variation }, vendorId: { type: String, // Id of the user uploading the product...Can be retried by using user._id - check user model for more info }, storeId:{ type: String, // Id of the store where the product is uploaded from. } }

NOTE: Whenever a product is added, a Post concerning that product is also created

GET
/products/:userId // Get all products belonging to a particular user --- userId here could also be called vendorId
Response // It returns an array of all products belonging to a user [ { _id, category, images, // Array of image urls description, price, totalQuantity, available, variations, reviews, vendorId, storeId, createdAt, updatedAt } ]

NOTE: Role is the role of the user. 0 is regular customer, 1 is clothier, 2 is tailor, 3 is Admin

Get
/product/:id
Response Returns an a single product { _id, category, images, // Array of image urls description, price, totalQuantity, available, variations, reviews, vendorId, storeId, createdAt, updatedAt }

NOTE: The profile picture response is returned if the user has a profile picture, it was not required at signup but the user can upload a profile picture to the edit user route
